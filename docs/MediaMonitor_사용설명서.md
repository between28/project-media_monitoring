# MediaMonitor 사용 설명서

작성일: 2026-03-13

## 1. 프로그램 목적

MediaMonitor는 보도자료 HWPX를 기준으로 검색 쿼리 초안을 만들고, 사용자가 이를 직접 보완한 뒤 관련 기사를 수집하는 데스크톱 앱입니다.

수집된 결과는 세션 단위로 저장되며, 기사 목록 CSV와 브리핑 초안 Markdown을 함께 생성합니다.

## 2. 실행 전 준비

- `dist/MediaMonitor` 폴더 전체를 같은 위치에 둡니다.
- `MediaMonitor.exe`를 실행합니다.
- 입력할 보도자료는 `HWPX` 형식을 권장합니다.

## 3. 기본 사용 순서

1. 상단의 `HWPX 선택` 버튼으로 보도자료 파일을 불러옵니다.
2. 앱이 자동으로 `검색 쿼리` 초안을 생성합니다.
3. 사용자가 `검색 쿼리`와 `핵심 키워드`를 직접 수정하거나 추가합니다.
4. `기사 검색` 버튼을 눌러 수집을 시작합니다.
5. 실행이 끝나면 `기사 목록 열기`, `브리핑 열기`, `세션 폴더 열기`로 결과를 확인합니다.

## 4. 검색 규칙

### 검색 쿼리

- 여러 줄 중 `1줄 이상` 충족되면 검색 조건을 만족합니다.
- 각 줄 내부는 `띄어쓰기 기준 모든 단어 포함` 규칙으로 판단합니다.
- 예를 들어 `중동 바이오헬스`는 기사 제목/요약에 `중동`과 `바이오헬스`가 모두 있어야 매치됩니다.

### 핵심 키워드

- 핵심 키워드를 비워두면 이 조건은 적용하지 않습니다.
- 핵심 키워드를 입력한 경우, 기사 제목/요약에 `모든 핵심 키워드`가 포함된 기사만 최종 수집합니다.

### 수집 기간과 수집량

- 보도자료 배포일 오전 10시(한국시간)를 시작점으로 하여 `D+3 23:59:59`까지의 기사를 추적합니다.
- 실제 실행 시점에서는 이 범위 안에서 `현재까지 보도된 기사`만 수집합니다.
- `RSS/sitemap`은 소스당 최신 `100건`까지 읽습니다.
- `Google News`는 쿼리별 최대 `50건`까지 읽습니다.

## 5. 결과물 저장 위치

실행 결과는 `sessions/<session_id>/` 아래에 저장됩니다.

- `config/`
  - 자동 생성 쿼리, 수동 수정값, 최종 설정값
- `outputs/references/`
  - 기사 목록 CSV/Markdown
- `outputs/briefings/`
  - 일자별 브리핑 Markdown
- `outputs/latest_reference_articles.csv`
  - 최신 기사 목록
- `outputs/latest_briefing.md`
  - 최신 브리핑

## 6. 로그와 히스토리

- `logs/app.log`
  - 실행 로그
- `logs/query_history.csv`
  - 실행할 때마다 누적되는 보도자료 파일명 + 검색 쿼리 기록

## 7. 사용 팁

- 자동 추출된 검색 쿼리는 초안이므로, 실행 전에 반드시 사람이 검토하는 것이 좋습니다.
- 기사 목록 CSV가 열려 있는 상태에서도 실행은 가능하며, 필요한 경우 다른 이름으로 저장됩니다.
- 동일한 보도자료를 다시 실행하면 같은 세션을 이어서 사용할 수 있습니다.
- 다른 보도자료를 실행하면 새로운 세션 폴더가 생성됩니다.

## 8. 한계와 주의사항

- 일부 언론사는 RSS/sitemap이 없거나 불안정하여 Google News 경유로만 수집될 수 있습니다.
- Google News에서 수집된 링크는 원문 링크가 아니라 Google News 래퍼 링크일 수 있습니다.
- 기사 본문 전체를 전수 조사하지 않기 때문에, 제목/요약 중심 수집의 한계로 일부 기사가 누락될 수 있습니다.
