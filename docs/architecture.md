# 아키텍처 노트

## 왜 Apps Script + Google Sheets인가

이번 MVP는 무료 운영, 빠른 배포, 낮은 운영 난이도가 우선입니다. Google Apps Script는 별도 서버 없이 스케줄 실행과 RSS 수집을 처리할 수 있고, Google Sheets는 비개발자도 바로 확인·수정할 수 있는 운영 저장소 역할을 합니다. 대변인실의 아침 브리핑 업무에서는 복잡한 플랫폼보다 "매일 05:30 전후로 안정적으로 돌아가고 사람이 바로 열어볼 수 있는 구조"가 더 중요합니다.

다만 실제 운영이 길어질수록 Apps Script 편집기와 시트 중심 구조는 누적 저장, 재분석, 로컬 테스트, 백업 측면에서 불편해집니다. 그래서 현재 저장소에는 같은 규칙을 따르는 `python_pipeline/` 경로도 함께 두었습니다.

## 자동화 범위

현재 자동화되는 단계는 다음과 같습니다.

1. RSS 및 Google News RSS 수집
2. `news_raw` 적재
3. 링크/제목/정규화 제목/유사 제목 기준 중복 판정
4. 정책 키워드 기반 관련도 점수 산정
5. 규칙 기반 프레임 분류
6. 중요도 랭킹 계산
7. `briefing_output`에 한국어 브리핑 초안 생성

## 사람 검토가 남는 부분

- 브리핑 초안의 최종 문안 확정
- 민감한 정치적 표현 조정
- 기사 원문 확인이 필요한 사안 검증
- 빠진 RSS 소스 추가
- 잘못 분류된 프레임 교정

즉, MVP는 "선별과 초안화"를 자동화하고, 최종 보고 책임은 사람이 갖는 구조입니다.

## 이후 확장 방향

### Python

Python 경로는 더 이상 단순한 확장 아이디어가 아니라, 실제 대체 실행 경로입니다.

현재 `python_pipeline/`이 제공하는 것:
- `RSS`, `Google News RSS`, `sitemap` 수집
- `SQLite` 누적 저장
- 중복 제거, 점수화, 프레임 분류, 랭킹
- 브리핑 텍스트 파일 생성

Python 중심 구조의 장점:
- 기준 시점 재분석이 쉬움
- 수집과 브리핑 실행을 외부 스케줄러로 분리하기 쉬움
- 로컬 개발, 테스트, 백업, 로그 확인이 편함
- Google Sheets 수동 복붙 없이 자동화 가능

즉, Apps Script는 무료 MVP 진입 경로이고, Python은 안정성과 편의성을 높이는 운영 경로입니다.

### GitHub Actions

GitHub Actions는 향후 다음 용도로 적합합니다.

- 정해진 시각에 Python 수집기 실행
- 결과 파일 산출 및 버전 이력 보관
- 설정 파일 검증
- 테스트 자동화

초기 MVP는 Sheets 중심이었지만, 현재는 Python CLI를 Actions에서 직접 실행하는 구조가 더 자연스럽습니다. 배치 실행, 산출물 업로드, 설정 검증, 테스트 자동화에 모두 잘 맞습니다.

### GDELT

GDELT는 해외 보도와 대규모 이벤트 탐지 확장에 적합합니다. 다만 MVP 단계에서는 구조가 과해질 수 있으므로 제외했습니다. 후속 버전에서는 다음처럼 붙일 수 있습니다.

- 해외 보도 보강용 입력원 추가
- 이슈 급등 탐지 보조 지표 활용
- 국내 RSS와 분리된 보조 시트 운영

핵심은 동일한 데이터 모델과 규칙을 유지하면서, 운영 난이도에 따라 Apps Script 또는 Python 중 더 적합한 실행 경로를 선택할 수 있게 하는 것입니다.
