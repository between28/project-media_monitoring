# 브리핑 템플릿

자동 생성 브리핑은 아래 구조를 따릅니다.

## 1. 총평

- 3~4문장
- 전체 보도량, 우세 프레임, 반복 서사, 위험 신호를 요약

예시 형식:

```text
금일 오전 모니터링 기준, 도심 주택공급 확대 및 신속화 방안 관련 고관련 기사 12건이 선별되었습니다.
보도 흐름은 정책 설명 중심이나, 일부 매체에서는 실효성·사업 속도·지역 반응을 둘러싼 우려를 병행 제기했습니다.
반복적으로 등장한 서사는 용산 국제업무지구 공급, 태릉CC 공급, 후속 일정 관리였습니다.
브리핑 시에는 공급 규모뿐 아니라 집행 일정과 협의 절차를 함께 설명할 필요가 있습니다.
```

## 2. 주요 보도 내용

- 2~4개 주제로 묶어서 정리
- 각 주제별 대표 기사와 매체 분포 제시

예시 형식:

```text
- 용산·도심 공급 관련 보도: 연합뉴스, 한국경제 등 4건. 대표 기사: [연합뉴스] ...
- 태릉CC 후속 공급 관련 보도: 서울경제, 뉴스1 등 3건. 대표 기사: [서울경제] ...
```

## 3. 주요 논점

- 3~5개
- 프레임과 쟁점을 짧게 정리

예시 형식:

```text
- 공급 속도와 실제 착공 시점 간 괴리 가능성 제기
- 지역 수용성 및 관계기관 협의 필요성 부각
- 공급 확대 효과에 대한 시장 안정 기대 병존
```

## 4. 영향력 기사

- 상위 중요도 기사 3~5건
- 매체명, 제목, 프레임, 중요도 중심

예시 형식:

```text
1. [연합뉴스] ...
2. [한국경제] ...
3. [서울경제] ...
```

## 5. 대응 참고

- 대변인실 답변 포인트 3~4개
- 정책 설명 보완, 우려 대응, 기관 협의 메시지 중심

예시 형식:

```text
- 공급 물량과 일정은 확정안과 검토안을 구분해 설명
- 후속 인허가 및 관계기관 협의 절차를 구체 일정과 함께 제시
- 주민 우려 및 실효성 지적에는 단계별 이행관리 계획으로 대응
```

## 작성 원칙

- 기사 나열보다 반복 서사와 리스크 신호를 우선
- 제목/요약 수준에서 확인 가능한 사실만 사용
- 해석이 필요한 부분은 "가능성", "우려", "관찰" 수준으로 표현
