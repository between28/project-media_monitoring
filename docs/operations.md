# 운영 가이드

## 1. 기본 운영 흐름

권장 흐름:

1. 보도자료 `HWPX` 확보
2. 데스크톱 앱에서 파일 선택
3. 자동 생성된 `검색 쿼리` 확인
4. 필요 시 `검색 쿼리`, `핵심 키워드` 수정
5. `기사 검색` 실행
6. `기사 목록`과 `브리핑` 검토

## 2. 검색 규칙 원칙

### 검색 쿼리

- 자동 추출 초안입니다.
- 사용자가 추가, 변경, 삭제할 수 있습니다.
- 현재 후필터는 `띄어쓰기 기준 AND`로 동작합니다.
  - 예: `아일랜드 차관`
  - 제목/요약에 `아일랜드`와 `차관`이 모두 있으면 쿼리 일치로 봅니다.

### 핵심 키워드

- 수동 입력입니다.
- 값이 비어 있으면 핵심 키워드 조건은 적용하지 않습니다.
- 값이 입력되면 제목/요약에 모든 핵심 키워드가 포함된 기사만 최종 수집합니다.

## 3. 수집 범위

- 시작 시점: 보도자료 배포일 오전 10시(KST)
- 종료 시점: `D+3 23:59:59`
- 실제 실행에서는 위 범위 중 현재 시점까지의 기사만 수집합니다.

수집 입력원:
- `RSS`
- `sitemap`
- `Google News`

현재 기본 상한:
- RSS / sitemap: 소스당 최대 `100건`
- Google News: 쿼리당 최대 `50건`

## 4. 세션 폴더

실행 결과는 `sessions/<session_id>/` 아래에 저장됩니다.

주요 위치:
- `config/queries.auto.json`
- `config/queries.manual.ini`
- `config/config.effective.json`
- `data/session.sqlite3`
- `outputs/briefings/`
- `outputs/references/`

## 5. 기사 목록과 브리핑 차이

- 기사 목록: 참고자료용 전체 관련 기사 목록
- 브리핑: 본문에 사용할 대표 기사 중심 압축본

따라서 기사 목록 건수가 브리핑 본문 건수보다 많을 수 있습니다.

## 6. 중단과 재실행

데스크톱 앱에서:
- `기사 검색` 실행 중 `중단` 버튼을 누르면 협조적 취소가 걸립니다.
- 이미 진행 중인 네트워크 요청은 끝난 뒤 멈출 수 있습니다.
- 이후 `기사 검색`을 다시 누르면 처음부터 다시 실행합니다.

재실행 시 세션 DB의 실행 산출물 테이블은 비우고 다시 채웁니다.

## 7. 배포본 공유

배포 단위는 `MediaMonitor.exe` 하나가 아니라 폴더 전체입니다.

공유 방법:
1. `dist/MediaMonitor/` 또는 정리한 배포 폴더 전체 압축
2. 다른 사용자에게 전달
3. 상대방은 압축 해제 후 `MediaMonitor.exe` 실행

지워도 되는 파일:
- `logs/app.log`
- `logs/`
- `sessions/`
- 테스트용 입력 파일

남겨야 하는 파일:
- `MediaMonitor.exe`
- `_internal/`

## 8. 현재 제한

- `PDF-only` 보도자료는 아직 직접 파싱하지 않습니다.
- 일부 포털성 링크는 원문 URL 대신 래퍼 URL로 저장될 수 있습니다.
- 보도자료 제목 구조가 불규칙하면 자동 쿼리 초안 품질이 흔들릴 수 있습니다.
- 네이버 뉴스는 현재 수집원에 포함하지 않습니다.
